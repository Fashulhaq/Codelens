Drop files or a whole folder to scan
Everything is analyzed locally in your browser with the real engines — no upload, no install. Supports COBOL, Python, JS/TS, Java, CSS, SQL, dependency manifests, WSDL/XSD/XML, HTML, config files (.cfg/.ini/.properties/.env/.yaml/.toml), and Java archives (.war/.ear/.jar — unpacks & scans readable files + bundled libraries).
or scan a GitHub repository
Fetches files via the GitHub API — best for small/medium repos. For large repos, use the CLI scan_github.py (no rate limits).
Private repo on a company GitHub with SSO? Create a token (Settings → Developer settings → Personal access tokens), then — this step is easily missed — open the token and click Configure SSO → Authorize for your organization. Without that, the token is valid but blocked, and the scan will fail with an SSO message.

Runs the SAME real engines as Scan (via Pyodide) — full accuracy incl. real Python AST.
Every language runs its OWN engine, executed in your browser by Pyodide (CPython compiled to WebAssembly). Python uses the real ast module; SQL is parsed at statement level; JS/TS & Java use tokenizer + masking; CSS uses a block parser; COBOL uses the semantic control-flow engine. No file leaves your machine, nothing is installed. These are deep heuristic/AST engines, not full compilers — pair with language compilers and commercial SAST for mission-critical code.
